In Angular, the encapsulation property within the @Component decorator controls how styles are applied to the component's template. By default, Angular uses View Encapsulation to ensure that styles defined in a component only apply to that component's view and not to other components.

ViewEncapsulation.None

When you set encapsulation: ViewEncapsulation.None, Angular does not encapsulate the styles in the component. The styles defined in this component’s CSS will apply globally to the entire application, not just to the component’s view. You might use this in scenarios like: Global Styles, Third-Party Libraries, Custom Styling

ViewEncapsulation.Emulated

ViewEncapsulation.Emulated is the default mode of view encapsulation in Angular. Angular adds unique attributes to elements, ensuring styles are scoped to the component.
In this case, Angular adds the _ngcontent-c0 attribute to the h1 element, and applies the corresponding CSS to only the h1 within this component.

ViewEncapsulation.ShadowDom

ViewEncapsulation.ShadowDom: Uses the native Shadow DOM feature to scope styles, providing more encapsulation at the browser level.
In this case, the h1 styles only affect the elements within the Shadow DOM, and nothing outside of the Shadow DOM is influenced by these styles.
